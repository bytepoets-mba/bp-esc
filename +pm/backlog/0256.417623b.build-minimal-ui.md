Create simple UI for API key input and balance display.

## Tasks

- [ ] HTML structure (index.html)
  - API key input form
  - Balance display area
  - Buttons (Save, Refresh, Settings, Quit)

- [ ] CSS styling (style.css)
  - Clean, minimal design
  - BYTEPOETS branding (logo, colors)
  - Responsive layout
  - Error state styling

- [ ] JavaScript logic (app.js)
  - On load: call `read_api_key()`
  - If key exists: call `fetch_balance()`
  - If no key: show input form
  - Save button: call `save_api_key()` then fetch
  - Refresh button: call `fetch_balance()`
  - Quit button: call Tauri exit
  - Settings button: show input to update key

- [ ] Integrate Tauri commands
  - Import @tauri-apps/api
  - Call Rust backend functions

## Acceptance Criteria

- [ ] API key input works and saves
- [ ] Balance displays correctly (€ XXX.XX format)
- [ ] Refresh button fetches new balance
- [ ] Settings allows updating API key
- [ ] Quit button closes app
- [ ] Loading states show during async ops
- [ ] Errors display user-friendly messages
